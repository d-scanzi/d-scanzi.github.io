Phd Thesis with Bookdown Workflow

I recently completed my PhD thesis. I wrote the chapters as a series of R Markdown files and assemble it into a single PDF using Bookdown. I opted for Bookdown after experimenting with different options: Word, LogSeq to Word, manually knitting different R Markdown files, but nothing satisfied all the requirements I had. Namely:

  • Integration with Zotero: I did not want to manually format every inline reference and the full reference list.
  • Uses Latex: I did not want to manually place each figure and spend too much time formatting everything.
  • Deals seemlessly with long format writing: my thesis was about 200 pages
  • Programmatic query of data: I wanted to be able to report statistics, results and genrating images programmatically, by querying data rather than manually type every mean, standard deviation, etc...
  • Integration with R and Python: I needed to be able to run both R and Python code as all my analyses were done in either of those languages
  • Can be replicated: If I publish the data and the thesis, anyone can recompile the thesis and obtain the same result.
Bookdown ticks all these requirements and I could expand its capabilities further to obtain an even better final result. To set on this workflow, I pulled information from different sources:
  • B101info
  • nsaunder's R-bloggers post
  • Anna's post on Coding Club
  • Tyson Barret's blog
  • Mariana Montes' blog
  • Yihui Xie's Bookdown guide

However, there was no single guide containing all the details, and I had to do quite a bit of troubleshooting before getting something working properly. Thus, the reason for this post. A way to consolidate my workflow for future projects (e.g. journal articles) and to hopefully help someone else adopting a cool approach to write their dissertation. So, without further adue, let's get started.

Set up

For this guide, I will assume you are confortable using Zotero as a reference manager and RStudio as an IDE for R. If this is the very first time you use either of these software, you might want to take some time to understand them. There is plenty of information online and I might write a few guides about them in the future. Otherwise, we can start by setting up Zotero.

Set up: Zotero

At the time of writing I am using Zotero 7.0.16. The only one thing we really need is the Better Bibtex plugin. This plugin will help us with referencing material and keep the writing nice and tidy. So, Download and install the plugin. Now, we need to set up the Zoter directory from where we will pull the references. There is no strict rule regarding this, but this is my set-up. In My Library, I have a collection called Main were I store every paper I am interested in, independently of the project I am working on. I usually store papers using the Zotero connector or the DOI. I also created a second collection called phd_dissertation. Inside this collection I have a nested collection for each section of my thesis. Basically, my Zotero directory is the following:

My Library |-Main |-phd_dissertation | |_introduction | |_chapter_1 | |_chapter_2 | |_... | |_conclusions
